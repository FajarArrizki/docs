This page is the full tool-surface map for the current Rabit backend.

It answers four questions:

1. what tools exist right now
2. which family they belong to
3. where their value comes from
4. how failure is handled before the agent responds

## How Rabit handles tool failure

```mermaid
flowchart TD
    A[Agent selects tool] --> B[Tool registry validates parameters]
    B -->|invalid| C[Registry returns validation error details]
    B -->|valid| D[Tool executes]
    D -->|returns failure payload| E[Agent receives structured tool result]
    D -->|raises exception| F[Registry catches exception and normalizes error]
    E --> G[Agent adapts response]
    F --> G
    G --> H[Explain failure, suggest next step, or use fallback context]
```
