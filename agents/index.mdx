Rabit uses one concrete runtime agent: `TradingAgent`.

That choice is important.

Instead of introducing many separate specialist runtimes too early, Rabit keeps one adaptive entry point and makes it smarter through:

- intent routing
- tool grouping
- context handling
- exchange-aware behavior

## Why this matters

This keeps the architecture easier to reason about while still allowing the product to behave differently for:

- analysis
- monitoring
- execution-aware requests
- memory-backed conversations

## Why one agent is still enough today

The key point is that Rabit is not trying to win by multiplying runtime agents.

It is trying to win by making one agent:

- better informed
- better gated
- better connected to system tools
- safer around execution

That is a more disciplined architecture for the product stage Rabit is in now.
