## Current Charging Model

The current implementation charges a combined backend cost on-chain:

- `base_cost`: model/provider cost
- `service_cost`: monitoring or backend service cost

The contract combines them into one chargeable amount, then deterministically applies:

1. backend markup
2. platform fee
3. vault debit and usage receipt creation

This lets the product explain one settlement amount to the user while still preserving the split inside `AiUsageRecord`.
