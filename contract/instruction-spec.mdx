## Admin

| Instruction | Purpose |
| --- | --- |
| `initialize_config` | create global config and fee recipient metadata |
| `update_platform_fee` | update protocol fee |
| `update_default_markup` | update default markup |
| `update_authority` | transfer admin authority |
| `update_backend_authority` | update delegated charging backend |
| `toggle_pause` | pause or unpause charging |
| `claim_fees` | withdraw lamports held by the `fee_recipient` PDA |

## Spending Profile

| Instruction | Purpose |
| --- | --- |
| `initialize_spending_profile` | create a per-user charging profile |
| `approve_spending_delegate` | approve delegated SPL token spending |
| `revoke_spending_delegate` | revoke delegated SPL token spending |
| `close_spending_profile` | close the user's profile after delegated token approval is revoked |

## Delegation

| Instruction | Purpose |
| --- | --- |
| `create_delegated_signer` | create bounded backend charging authority |
| `revoke_delegated_signer` | disable delegated charging |
| `close_delegated_signer` | close revoked or expired delegation |

## Model Registry

| Instruction | Purpose |
| --- | --- |
| `register_model` | create model metadata entry as admin |
| `update_model` | change model metadata as admin |
| `deactivate_model` | disable a model entry as admin |

## Usage Charging

### record_ai_usage

Direct user-signed charging path.

Main inputs:

- `model_id`
- `base_cost`
- `service_cost`
- `usage_type`
- `tokens_used`
- optional `markup_bps`

Accepted `usage_type` values:

- `text`
- `image`
- `audio`
- `video`

Main accounts:

- `spending_profile`
- `user_token_account`
- `fee_recipient_token_account`
- `payment_mint`
- optional `model_registry`

Note: this charging path settles into a runtime-supplied SPL token account. That path is separate from `claim_fees`, which moves lamports out of the `fee_recipient` PDA.

### record_ai_usage_with_delegation

Delegated backend-signed charging path.

Main differences from direct path:

- requires `delegated_signer`
- requires backend signer to match `config.backend_authority`
- requires delegated spending limit to cover `total_charged`
- uses delegated SPL token spending path instead of direct user signature on transfer
- accepts only `text`, `image`, `audio`, or `video` as `usage_type`
