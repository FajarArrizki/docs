## Why some registry ids are shorter

The contract currently derives `ModelRegistry` PDAs from the on-chain `model_id` string.

Because a Solana PDA seed is limited to 32 bytes, some OpenRouter model ids are too long to use directly.

For those models, Rabit uses a deterministic shortened alias that:

- stays visually similar to the original id
- keeps the provider prefix
- adds a short hash suffix for uniqueness

Backend-side lookup can use:

- alias map file: `target/model-registry-aliases.json`
- reusable helper: `scripts/model-registry-helper.js`

That allows off-chain code to translate cleanly between provider ids and on-chain registry ids.
