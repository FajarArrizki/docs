## Quick Start

### 1. Build Docker Image
```bash
make docker-build
# or
docker-compose build
```

### 2. Start Development Environment
```bash
make docker-up
# or
docker-compose up -d
```

### 3. Access Container Shell
```bash
make docker-shell
# or
docker-compose exec solana-dev /bin/bash
```

### 4. Inside Container - Build & Test
```bash
# Build the program
anchor build

# Run tests
anchor test

# Deploy to local validator
anchor deploy
```

## Docker Commands

### Start Services
```bash
docker-compose up -d
```

### Stop Services
```bash
docker-compose down
```

### View Logs
```bash
docker-compose logs -f
```

### Rebuild Image
```bash
docker-compose build --no-cache
```

### Remove All Data
```bash
docker-compose down -v
```

## What's Included

The Docker image includes:
-  Ubuntu 22.04 base
-  Rust (latest stable)
-  Solana CLI v1.18.26
-  Anchor CLI v0.30.1
-  Node.js 18
-  Yarn package manager

## Ports Exposed

- `8899` - Solana RPC
- `8900` - Solana RPC Pubsub
- `9900` - Solana Faucet

## Volumes

- `./` → `/workspace` - Your project files
- `solana-data` - Solana validator data (persistent)
- `cargo-cache` - Rust cargo cache (persistent)

## Development Workflow

### 1. Start Environment
```bash
make docker-up
```

### 2. Enter Container
```bash
make docker-shell
```

### 3. Inside Container
```bash
# Configure Solana
solana config set --url localhost

# Generate keypair
solana-keygen new

# Build program
anchor build

# Run tests
anchor test

```

### 4. Stop Environment
```bash
make docker-down
```

## Troubleshooting

### Container won't start
```bash
# Check logs
docker-compose logs

# Rebuild
docker-compose build --no-cache
docker-compose up -d
```

### Permission issues
```bash
# Fix file permissions
sudo chown -R $USER:$USER .
```

### Port already in use
```bash
# Check what's using the port
lsof -i :8899

# Kill the process or change port in docker-compose.yml
```

### Clean slate
```bash
# Remove everything and start fresh
docker-compose down -v
docker-compose build --no-cache
docker-compose up -d
```

## Tips

1. **Persistent Data**: Validator data and cargo cache are stored in Docker volumes, so they persist between container restarts.

2. **Hot Reload**: Your code changes are immediately reflected in the container since the project directory is mounted.

3. **Multiple Terminals**: You can open multiple shells to the same container:
   ```bash
   docker-compose exec solana-dev /bin/bash
   ```

4. **Copy Files**: Copy files from container to host:
   ```bash
   docker cp rabit-solana-dev:/workspace/target/deploy/rabit_contract.so ./
   ```

5. **Resource Limits**: If the validator is slow, increase Docker's memory allocation in Docker Desktop settings.

## CI/CD Integration

### GitHub Actions Example
```yaml
name: Test
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Build and Test
        run: |
          docker-compose build
          docker-compose up -d
          docker-compose exec -T solana-dev anchor test
```
