## Main Components

| Component | Scope | Purpose |
| --- | --- | --- |
| `PlatformConfig` | Singleton | Stores authority, fee policy, pause state, backend authority, and fee recipient metadata |
| `SpendingProfile` | Per user | Stores payment mint plus cumulative charging totals |
| `DelegatedSigner` | Per user and delegate pair | Allows bounded backend spending without requiring a user signature for every usage record |
| `AiUsageRecord` | Per usage | Creates an immutable receipt for billing, analytics, and auditing |
| `ModelRegistry` | Per model | Stores model metadata, optional pricing hints, and usage counters |

## Design Principles

| Principle | Why it exists |
| --- | --- |
| user funds remain in user-owned token accounts | avoids forced prepaid contract custody |
| spending is delegated explicitly | backend automation stays bounded by user approval |
| profile state is separate from payment accounts | policy and accounting can evolve without becoming custody |
| usage is recorded immutably | every accepted charge has an on-chain explanation |

## Features Breakdown

### 1. Admin Management

| Instruction | Purpose |
| --- | --- |
| `initialize_config` | one-time platform setup |
| `update_platform_fee` | change platform fee |
| `update_default_markup` | change default backend markup |
| `update_authority` | transfer admin control |
| `update_backend_authority` | change backend wallet |
| `toggle_pause` | stop or resume charging |
| `claim_fees` | withdraw lamports held by the `fee_recipient` PDA |

### 2. Model Registry

| Instruction | Purpose |
| --- | --- |
| `register_model` | add AI model to registry |
| `update_model` | update pricing hint or metadata |
| `deactivate_model` | disable model usage |

Long model ids:

- some OpenRouter ids are longer than Solana's 32-byte PDA seed limit
- those models use a deterministic shortened on-chain alias
- the mapping is exported to `target/model-registry-aliases.json`

### 3. Spending Profile

| Instruction | Purpose |
| --- | --- |
| `initialize_spending_profile` | create user charging profile |
| `approve_spending_delegate` | approve delegated SPL token spending |
| `revoke_spending_delegate` | revoke delegated SPL token spending |
| `close_spending_profile` | close user profile and reclaim rent after delegate approval is cleared |

### 4. Delegated Signers

| Instruction | Purpose |
| --- | --- |
| `create_delegated_signer` | create bounded backend charging authority |
| `revoke_delegated_signer` | instantly disable delegation |
| `close_delegated_signer` | close revoked or expired delegation |

### 5. AI Usage Recording

| Instruction | Purpose |
| --- | --- |
| `record_ai_usage` | direct user-signed token charging |
| `record_ai_usage_with_delegation` | delegated backend-signed token charging |

Security notes:

- `register_model`, `update_model`, and `deactivate_model` are admin-gated through `config.authority`
- `record_ai_usage*` only accepts `usage_type` values in the built-in set: `text`, `image`, `audio`, `video`
- `close_spending_profile` now requires the user's token delegate approval to already be revoked

Charging settlement note:

- usage charging does not transfer SPL tokens into the `fee_recipient` PDA directly
- runtime charging instead uses a `fee_recipient_token_account` supplied with the instruction
- the program requires that token account to be owned by `config.authority`
- `claim_fees` remains a separate lamport withdrawal path for the PDA itself

## Fee Structure

### Fee Calculation Formula

```text
chargeable_cost = base_cost + service_cost
markup_amount = chargeable_cost Ă— markup_bps / 10000
cost_after_markup = chargeable_cost + markup_amount
platform_fee = cost_after_markup Ă— platform_fee_bps / 10000
total_charged = cost_after_markup + platform_fee
```

### Example Calculation

| Input | Value |
| --- | --- |
| base_cost | 1000 |
| service_cost | 200 |
| markup_bps | 500 |
| platform_fee_bps | 500 |

```text
chargeable_cost = 1200
markup_amount = 60
cost_after_markup = 1260
platform_fee = 63
total_charged = 1323
```

### Breakdown

- user pays `1323` units of the configured SPL token
- model cost = `1000`
- service cost = `200`
- markup = `60`
- platform fee = `63`
