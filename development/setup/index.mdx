This section is for contributors working on the backend codebase.

## Recommended Flow

1. Read the public docs first:
   - [Getting Started](../../getting-started)
   - [Architecture](../../architecture)
2. Run the backend locally
3. Run targeted tests for the area you change
4. Update documentation when behavior changes

## Development Areas

- agent runtime
- tools
- integrations
- WebSocket and market data
- authenticated execution flows

## Documentation Rule

Public behavior should be documented in the main docs tree.

Historical implementation notes, checklists, and migration records should stay outside the public Mintlify surface.
