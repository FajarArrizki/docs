This section is for contributors working on the Rabit backend.

It is intentionally narrower than the product-facing sections. The goal here is to make active work on the codebase easier.

## What belongs here

| Topic | Why it belongs here |
| --- | --- |
| setup and workflow | helps contributors get the backend running |
| local development expectations | keeps day-to-day work predictable |
| mobile-facing data requirements | helps contributors understand the client-facing contract |

## Why this still matters

Rabit is also a living codebase, not only a demo surface.

Good development documentation shows that the project has:

| Engineering signal | Why it matters |
| --- | --- |
| contributor workflow | the codebase can actually be worked on |
| environment expectations | setup is documented instead of assumed |
| a path to iterate on the backend | the project is built to keep moving |
